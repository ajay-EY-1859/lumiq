Build me a provider-agnostic coding agent CLI called OpenClaude. It's a terminal-first tool that lets developers run agentic LLM workflows across multiple cloud and local providers without changing their workflow. Think of it as Claude Code but with the Anthropic lock-in removed.

The core experience is an interactive REPL rendered in the terminal using React 19 and Ink. Users type prompts, the agent reasons about them, calls tools (bash, file read/write/edit, grep, glob, web search), and streams responses back — all inside the terminal. It should also support a non-interactive headless mode via a --print flag for scripting and CI.

For providers, support Anthropic, OpenAI (and OpenAI-compatible endpoints), Google Gemini, GitHub Models, AWS Bedrock, Google Vertex AI, Ollama, and LM Studio. Select the active provider via environment variables like CLAUDE_CODE_USE_OPENAI=1 or CLAUDE_CODE_USE_GEMINI=1, with the corresponding API key and optional base URL. For any provider that requires a real API key, include a working stub or mock implementation so the app runs immediately without configuration. Save per-project provider settings in a .openclaude-profile.json file at the project root that stores provider, baseUrl, apiKey, and model. Use Bun as the runtime, bundler, and package manager.

The agent loop works like this: user submits a message, the query engine builds a system prompt and tool list, calls the provider API with streaming, receives text deltas and tool invocations, executes tools, appends results to context, and loops until the model signals end_turn or a max turn limit is hit. Persist sessions as JSONL files at ~/.claude/projects/<cwd>/<session-id>.jsonl, one line per message or tool event. Support resuming a past session with /resume and starting fresh with /new.

Implement at least these tools: Bash (shell execution with an allowlist/denylist), FileRead, FileWrite, FileEdit (unified diff application), Glob, Grep (ripgrep-backed, respects .gitignore), WebSearch (default to DuckDuckGo with no key required, with optional Tavily, Exa, and Firecrawl backends controlled by env vars), and WebFetch (fetch a URL and return markdown). Add a tool permission system with three modes — manual (prompt for each tool use), auto (approve based on classifier), and bypass (no prompts). Track denials per session so the same tool call isn't re-prompted on retry.

Build the TUI with these visual details. Use chalk and Ink components. The layout has a status line at the top showing model name and session cost, a scrollable message list in the middle, and an input area at the bottom with a footer showing key bindings. Keyboard shortcuts: Ctrl+C to exit, Ctrl+U to clear input, Ctrl+T to toggle a tasks sidebar, Enter to submit, Tab to autocomplete slash commands. Slash commands to implement: /model, /new, /resume, /context, /provider, /help, /compact. Support dark and light themes. Use these exact RGB values for UI elements: assistant messages rgb(215,119,87), permission prompts rgb(87,105,247), success rgb(44,122,57), error rgb(171,43,63), warning rgb(150,108,30). Use box-drawing characters for borders. Add a spinner with shimmer for in-progress tool calls, and respect a prefersReducedMotion setting.

For configuration, load settings in this priority order: CLI flags, then .openclaude-profile.json, then environment variables, then ~/.claude/settings.json. Settings file is JSONC (supports comments). Key settings: model, permissionMode, webSearchProvider, theme. Support hot-reload of the settings file using a file watcher.

Also build a headless gRPC server mode that exposes a bidirectional Chat stream so other tools can drive the agent programmatically. Session IDs allow reconnecting to existing sessions. Tool approval prompts are sent as events over the stream so the caller can respond.

For context window management, track token usage per turn, apply prompt caching where the provider supports it, and auto-compact old messages when approaching the context limit by summarizing earlier turns. Count extended thinking tokens separately if the model supports it.

Parse CLI arguments with Commander.js. Use Zod for all config and tool argument validation. Use Bun's test runner for unit tests. For error handling, map provider-specific errors to friendly messages ("Local provider not running — please start Ollama", "Model not found — use /model to switch", etc.). Retry on rate limits with exponential backoff, defaulting to 10 retries, with an env var CLAUDE_CODE_UNATTENDED_RETRY=1 for infinite retries in CI. Include a CLAUDE_DEBUG=1 env var for verbose stderr logging.
