# 🛡️ Security Policy

We at **Fairline tech** take the security of **Lumiq** very seriously. If you believe you have discovered a security vulnerability or exploit in the codebase, we ask that you report it to us privately so we can patch the issue before it is publicly disclosed.

---

## 📅 Supported Versions

We actively provide security patches and updates for the following versions of Lumiq:

| Version | Supported | Notes |
| :--- | :--- | :--- |
| **v1.x** | ✅ Yes | Current Stable & Active Development Release |
| **< v1.0.0** | ❌ No | Legacy pre-release scaffolding versions |

---

## 🚨 Reporting a Vulnerability

Please **do not** open public GitHub issues for security-sensitive bugs or exploits. Public disclosure can expose active user environments to risk before a patch is ready.

### Steps to Report:
1. Gather all relevant information about the security bug, including:
   - A detailed description of the vulnerability.
   - Step-by-step instructions (or a proof-of-concept script) to reproduce the exploit.
   - The impact of the exploit (e.g. local privilege escalation, remote code execution, keystore leak).
2. Report the vulnerability privately:
   - Contact **Ajay Raj** directly via private channels or open a confidential report if your platform supports it. You can check the maintainer's profile at [@ajay-EY-1859](https://github.com/ajay-EY-1859) to find direct contact details or open a discussion thread requesting private security communication.
3. We will acknowledge receipt of your vulnerability report within **48 hours** and work on a prompt resolution.

---

## 🛠️ Handling of Vulnerability Reports

Once a security issue is reported:
1. **Validation:** We will reproduce and validate the reported security exploit.
2. **Patch Development:** A patch will be developed in a private branch to resolve the issue securely.
3. **Release & Patch Backport:** The fix will be merged into the master repository and a new release will be published immediately.
4. **Public Disclosure:** We will publish a security advisory alongside the release, giving proper credit to the reporter for their discovery if they wish.

Thank you for helping keep **Fairline tech** products safe for developers worldwide!
